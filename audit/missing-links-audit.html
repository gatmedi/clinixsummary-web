6. Contact Form — mailto: Must Become Online Form  HIGH

The contact form currently constructs a mailto: link instead of submitting data to a server. This must be replaced with an online form that submits server-side and delivers the email.

File : LineCurrent Behaviour
js/pages/other.js : 1014 The <form id="contact-form"> renders fields for Name, Email, Organisation, Topic (dropdown), and Message.
js/pages/other.js : 1054–1074 The submitContactForm() function:
• Reads form values
• Chooses enterprise@clinixsummary.ai or contact@clinixsummary.ai based on topic
• Constructs a mailto: link and opens it via window.location.href
• This depends on the user having a desktop email client — fails silently on many mobile devices and webmail-only users
Developer action: Replace the mailto: logic with an actual form submission. Options:
• Formspree / Formcarry / Web3Forms — third-party service, no backend needed. Just change the form action attribute and method to POST.
• Custom API endpoint — e.g. POST /api/contact that sends the email via SendGrid, SES, or similar.
• Console backend — if the console app already has an API, add a contact endpoint there.

The form should: (1) submit data via AJAX/fetch, (2) show a success confirmation on the page, (3) deliver the email to enterprise@ or contact@ based on topic.
